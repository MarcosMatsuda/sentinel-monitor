SentinelMonitor roadmap

Multi-camera B2B dashboard. WebRTC P2P, pareamento permanente, grid 1×1 / 2×2 / 3×3, Expo Universal (iOS / Android / Web). Caminho até v1.0 e além.

Fase 1 — MVP v1.0
~10 dias · entrega via 11 PRs automáticos

Pareamento permanente por código

Câmera mostra código de 6 chars (TTL 5min). Dashboard digita uma vez. Bind salvo dos dois lados — reconecta sozinho em qualquer reabertura.

UUID v4localStorageAsyncStorage

WebRTC P2P por UUID

Server roteia signal entre peers por UUID, não por sala. Sem DB — clientes são source of truth dos bindings.

WebRTCSocket.IOPresence map

Grid auto 1x1 / 2x2 / 3x3

Layout calculado pelo número de câmeras conectadas. 1 câm = fullscreen, 2-4 = 2x2, 5-9 = 3x3. Tap em uma tile abre fullscreen.

Expo RouterFlexboxZustand

Renomear, remover, mute por câmera

Apelido por câmera ('Sala', 'Cozinha'). Botão de mute por tile. Remover binding apaga local sem afetar a câmera.

ZustandPersistent storage

Indicador online/offline ao vivo

Server emite presence-change. Tile mostra dot verde (online), âmbar (reconectando), vermelho (offline). Atualiza em tempo real.

Presence subscribeWebRTC ICE

Signaling server stateless

Node.js + Socket.IO. In-memory presence + pairing codes. Sem DB no MVP. Reconverge em 1 ciclo de reconnect após restart.

ExpressSocket.IORender.com

Viewer Universal (iOS / Android / Web)

Uma codebase Expo, 3 plataformas. Platform.select via file resolution (.web.tsx vs .native.tsx) para WebRTC e storage.

Reconnect harden

Fase 2 — AI cross-camera tracking v2
2-3 semanas após v1.0 · o diferencial técnico do produto

Detecção de objetos por câmera

YOLO ou RT-DETR rodando on-device (TensorFlow.js / WebGPU no browser). Classes: pessoa, veículo, sacola. Bounding boxes overlay no tile.

TFLite / TFJSWebGPUYOLO v8

Watchlist de suspeitos

Operador marca uma pessoa/veículo no tile como suspeito. Sistema gera embedding visual (ReID) e adiciona à watchlist da sessão.

ReID embeddingAnthropic Vision

Cross-camera alert

Quando detecção em qualquer outra câm faz match com embedding da watchlist (acima de threshold), dashboard pisca alerta + tile destacado.

Cosine similarityServer fanout

License plate recognition (LPR)

Para veículos: extrai placa via OCR (Tesseract WASM ou cloud). Watchlist de placas com match exato, sem embedding.

Tesseract.jsPlate Recognizer

Timeline de detecções

Botão no dashboard abre timeline lateral com todas as detecções da sessão. Filtro por câmera, classe, intervalo.

Virtual listIndexedDB

Push notifications

Alerta cross-camera dispara push para o app nativo do operador, mesmo com app fechado. FCM (Android) + APNs (iOS).

FCMAPNsExpo Notifications
Fase 3 — Escala & produto v3
Quando virar produto real (clientes pagantes)

Multi-tenant + auth

Clerk ou Supabase Auth. Cada empresa tem suas próprias câmeras + dashboards. Isolamento por tenant no server.

ClerkSupabaseJWT

Recording + storage

Gravação contínua ou triggered (motion / detection). Storage em Cloudflare R2 com retention policies. Player com timeline scrubbing.

MediaRecorderR2HLS playback

SFU para escala (mediasoup)

Sair de mesh quando viewer tiver mais de 9 câmeras ou múltiplos viewers por tenant. SFU central reduz CPU/banda no mobile.

mediasoupNode SFU

Motion zones configuráveis

Operador desenha polígonos no frame indicando áreas de interesse. Detecções fora da zona não disparam alerta.

Canvas drawingPolygon-in-bbox

Suporte a IP cameras (RTSP)

Bridge RTSP → WebRTC (via go2rtc ou Janus). Cliente final pluga câmera de segurança tradicional, dashboard vê via mesmo grid.

go2rtcJanusRTSP

Relatórios + dashboards analíticos

Página de analytics: detecções por câmera/dia, horários de pico, alertas por tipo. Export PDF/CSV.

PostgreSQLRecharts
